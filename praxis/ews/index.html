Architecture

How It Works

TREMOR is a three-stage pipeline: ingest, model, alert. Each stage is automated and runs at weekly, district-level resolution across 13 countries.

1. Data Ingestion

The system pulls event-level data from the ACLED API on a weekly cycle. Each record is geocoded to admin-2 level and tagged by event sub-type.

2. Econometric Modelling

A composite co-occurrence detector flags district-weeks where two or more distinct conflict event sub-types converge. The primary validated signal is KAFD + IED co-occurrence, which produces an 8.8x escalation probability relative to the regional baseline. The detection window is seven days forward.

3. Structured Alerts

When a signal crosses threshold, TREMOR generates a structured alert. Each alert carries four fields: severity (Watch / Elevated / High / Critical), geographic target (admin-2 district), trigger (the specific event type combination), and baseline comparison.

Platform Layers

Layer 1: Predict

Composite co-occurrence detection from ACLED API data at weekly resolution. The spike detector triggers on convergence of 2+ distinct conflict event sub-types in the same admin2-week. The primary signal is KAFD + IED co-occurrence (8.8x baseline). The output is an escalation alert at admin2 level with a 7-day forward window.

Layer 2: Alert

Signal translation into structured alerts for defined stakeholder groups. Each alert includes:

Severity
Watch / Elevated / High / Critical
Targeting
Geographic precision to district level

Layer 3: Respond (Roadmap)

This layer is not yet built. The plan is to attach decision pathways to each alert tier so that different users receive guidance matched to their mandate. What that looks like in practice:

A P/CVE implementer running community dialogue in Tillaberi gets an alert that says "pause planned field activities in these three communes for seven days." A humanitarian logistics officer in Mopti gets a prompt to verify pre-positioned stock levels in the flagged corridor. A security coordination body gets a structured threat brief they can fold into their weekly sitrep.

None of this exists yet. Right now, TREMOR produces the alert. What users do with it is on them.

The Sahel is the proof of concept. The model works anywhere ACLED does.

The co-occurrence detection logic is not region-specific. It runs on event sub-type convergence patterns, which ACLED codes consistently across all its coverage areas. The Sahel is where the 8.8x signal was validated, but the same pipeline can deploy to any ACLED-covered context without retraining. That's currently 13 countries. ACLED covers roughly 230.

Planned Extensions

Three additions are scoped but not yet in development: XGBoost classification on multi-source event features to improve signal precision beyond the current rule-based detector; automated actor-network extraction to identify which armed groups are co-present in flagged district-weeks; and deep temporal sequence modelling to extend the forward window beyond seven days.
